# Google 代码审查标准

## 审查步骤

### 1. 代码风格
- 检查是否符合项目代码风格指南
- 变量命名是否清晰有意义
- 函数和类的命名是否准确
- 缩进和格式是否一致

### 2. 安全性检查
- 是否有 SQL 注入风险
- 是否有 XSS 漏洞
- 是否正确处理用户输入
- 敏感数据是否加密存储
- 是否有权限检查

### 3. 性能优化
- 是否有不必要的循环
- 数据库查询是否优化
- 是否正确使用缓存
- 是否有内存泄漏风险

### 4. 最佳实践
- 错误处理是否完善
- 日志记录是否充分
- 测试覆盖率是否足够
- 文档是否完整

## 审查输出格式

对于每个问题，请提供：
1. **问题类型**：风格/安全/性能/最佳实践
2. **严重程度**：高/中/低
3. **具体位置**：文件名和行号
4. **问题描述**：详细说明问题
5. **建议修复**：提供具体的修复方案

## 示例

```
问题类型：安全性
严重程度：高
具体位置：user_auth.py:45
问题描述：SQL 注入风险 - 直接拼接用户输入到 SQL 查询
建议修复：使用参数化查询或 ORM 框架
```
